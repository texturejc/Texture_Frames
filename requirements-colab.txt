# Canonical pinned environment for this project on Google Colab.
# Covers BOTH baseline reproduction (T5) and Path B encoder training (DeBERTa-v3).
#
# Install with:
#     pip install -r requirements-colab.txt
#
# Two deliberate non-pins that keep Colab stable:
#
#   1. torch / torchvision / torchaudio are NOT listed. Use the CUDA-matched
#      build that already ships on the Colab GPU image. Reinstalling torch is the
#      #1 cause of broken CUDA on Colab.
#
#   2. protobuf is NOT pinned. Defeat the protobuf C++ backend error at runtime
#      by setting this BEFORE importing transformers / sentencepiece:
#          import os
#          os.environ["PROTOCOL_BUFFERS_PYTHON_IMPLEMENTATION"] = "python"
#      (The notebooks do this for you.)
#
#   3. numpy IS pinned to >=2. Colab (2025+) ships numpy 2.x and its torch /
#      transformers are built against it. A stray `pip install numpy` or an old
#      transformers cap can silently downgrade numpy to 1.x, which then breaks
#      everything with either "numpy.dtype size changed" or
#      "No module named 'numpy.strings'". Pinning >=2 makes `pip install`
#      *repair* such a downgrade. (If you hit this on a dirty runtime, you must
#      also Delete the runtime — Restart session keeps the bad numpy on disk.)

# --- core: baseline reproduction + inference ---
numpy>=2                   # keep consistent with Colab's numpy-2 stack (see note 3)
transformers>=4.44         # numpy-2 compatible; do NOT cap below this
sentencepiece>=0.1.99      # required by both T5 and DeBERTa-v3 tokenizers
nltk>=3.8                  # FrameNet 1.7 corpus + WordNet

# --- training (Milestone 2+) ---
accelerate>=0.30          # required by HF Trainer

# Deliberately NOT included, to keep the compiled-dependency surface (and thus
# numpy-ABI conflicts on Colab) minimal:
#   * datasets / pyarrow  — we feed the Trainer a plain torch Dataset instead
#     (encoder_parser/data.build_trigger_dataset).
#   * seqeval / evaluate  — we compute P/R/F1 ourselves (encoder_parser/data.py,
#     eval_trigger.py) with the exact baseline-comparable definition.
# Add them back only if a later slice genuinely needs them.

# nlpaug is intentionally NOT listed here. The vendored parser's augmentation
# modules import it at load time, so the *baseline* notebook (which imports the
# full loaders) needs it — but installing it normally drags in a numpy/pandas
# that breaks Colab's ABI ("numpy.dtype size changed"). The baseline notebook
# therefore installs it separately with `pip install --no-deps nlpaug`.
# The encoder pipeline (encoder_parser/) avoids nlpaug entirely: its FrameNet
# loader (data.load_trigger_sentences) reads the corpus directly and never
# imports the augmentation modules.
